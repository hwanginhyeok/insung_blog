# W2: 인증 + 대시보드

> Supabase Auth 기반 로그인 + 내 글 관리 대시보드
> 작업 디렉토리: `apps/web/`
> 의존성: W1 완료 후 착수

---

## W2-01: Supabase Auth 로그인/회원가입 페이지

**목표**: 이메일/비밀번호 인증

**작업 내용**:
1. `app/login/page.tsx` — 로그인 폼 (이메일 + 비밀번호)
2. `app/signup/page.tsx` — 회원가입 폼
3. Supabase Auth 연동 (`signInWithPassword`, `signUp`)
4. 로그인 후 `/dashboard`로 리다이렉트
5. 에러 핸들링 (잘못된 비밀번호, 이미 존재하는 이메일 등)
6. MVP에서는 인성이만 사용 → 회원가입은 비공개(직접 가입)도 고려

**UI 컴포넌트**: shadcn/ui Input, Button, Card (이미 설치됨)
**완료 기준**: 로그인 → 세션 생성 → 대시보드 접근 가능

---

## W2-02: 미들웨어 — 비로그인 시 리다이렉트

**목표**: 보호된 라우트 설정

**작업 내용**:
1. `middleware.ts` 생성 (프로젝트 루트)
2. Supabase Auth 세션 확인
3. 비로그인 시 → `/login` 리다이렉트
4. 보호 경로: `/dashboard`, `/write`, `/settings`
5. 공개 경로: `/`, `/login`, `/signup`

**완료 기준**: 비로그인 상태에서 `/dashboard` 접근 시 `/login`으로 이동

---

## W2-03: 대시보드 레이아웃

**목표**: 인증된 사용자용 기본 레이아웃

**작업 내용**:
1. `app/(dashboard)/layout.tsx` — 공통 레이아웃
   - 상단 헤더: 로고 + 사용자 이름 + 로그아웃
   - 사이드 네비게이션 (또는 탭): 내 글, 새 글 쓰기, 설정
2. `components/nav/` — 네비게이션 컴포넌트

**완료 기준**: 대시보드 레이아웃 렌더링 + 로그아웃 동작

---

## W2-04: 내 글 목록 페이지

**목표**: generation_queue에서 내 글 조회

**작업 내용**:
1. `app/(dashboard)/dashboard/page.tsx` — 글 목록
2. generation_queue 테이블에서 user_id로 필터
3. 카드 형태로 표시:
   - 제목 (generated_title)
   - 상태 뱃지 (pending/processing/completed/failed)
   - 카테고리
   - 생성일
4. 상태별 필터 탭
5. 빈 상태 UI ("아직 작성한 글이 없습니다" + 새 글 쓰기 버튼)

**완료 기준**: generation_queue 데이터가 목록으로 렌더링됨
