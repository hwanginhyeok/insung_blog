# W5: 페르소나 학습 파이프라인

> 블로그 URL → 크롤링 → AI 분석 → DB 페르소나 관리 + HTML 렌더링 + 피드백 루프
> 설계 확정: 2026-03-07

## 개요

기존 PRODUCTION_SPEC.md (하드코딩 파일) → **DB 기반 페르소나 + HTML 렌더러**로 전환.
사용자별 글쓰기 스타일을 학습하고, 텍스트뿐 아니라 HTML 포맷팅(폰트, 사이즈, 줄간격, 볼드, 정렬)까지 재현한다.

### 핵심 결정 사항

| 항목 | 결정 |
|------|------|
| 크롤링 방식 | 모바일 URL (`m.blog.naver.com`) + cheerio (서버리스 호환) |
| AI 분석 모델 | Sonnet (1회성 분석, 정확도 우선) |
| 페르소나 관계 | 1 user : 1 persona (UNIQUE 제약) |
| 항목 카테고리 | voice, emoji, structure, ending, forbidden, custom, **formatting** (7개) |
| 항목 출처 | ai (크롤링), user (직접 추가), feedback (피드백 도출) |
| 피드백 규칙 적용 | 자동 아닌 사용자 승인 후 적용 |
| HTML 렌더링 | formatting 항목 기반 SmartEditor 호환 HTML 생성 → 리치텍스트 복사 |

## 데이터 모델

### user_personas

```sql
CREATE TABLE user_personas (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL UNIQUE REFERENCES auth.users(id) ON DELETE CASCADE,
  display_name TEXT NOT NULL DEFAULT '',
  source_blog_url TEXT,
  crawl_status TEXT DEFAULT 'none'
    CHECK (crawl_status IN ('none', 'crawling', 'analyzing', 'done', 'error')),
  crawl_post_count INT DEFAULT 0,
  crawled_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### persona_items

```sql
CREATE TABLE persona_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id UUID NOT NULL REFERENCES user_personas(id) ON DELETE CASCADE,
  category TEXT NOT NULL
    CHECK (category IN ('voice', 'emoji', 'structure', 'ending', 'forbidden', 'custom', 'formatting')),
  key TEXT NOT NULL,
  value TEXT NOT NULL,
  priority INT DEFAULT 0,
  is_active BOOLEAN DEFAULT TRUE,
  source TEXT NOT NULL DEFAULT 'ai'
    CHECK (source IN ('ai', 'user', 'feedback')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_persona_items_persona ON persona_items(persona_id);
CREATE INDEX idx_persona_items_active ON persona_items(persona_id, is_active) WHERE is_active = TRUE;
```

### persona_feedback

```sql
CREATE TABLE persona_feedback (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id UUID NOT NULL REFERENCES user_personas(id) ON DELETE CASCADE,
  generation_id UUID REFERENCES generation_queue(id) ON DELETE SET NULL,
  feedback_text TEXT NOT NULL,
  derived_rule TEXT,
  rule_status TEXT DEFAULT 'pending'
    CHECK (rule_status IN ('pending', 'approved', 'rejected')),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_persona_feedback_persona ON persona_feedback(persona_id);
```

### RLS 정책

```sql
-- user_personas: 본인 데이터만
ALTER TABLE user_personas ENABLE ROW LEVEL SECURITY;
CREATE POLICY "사용자 본인 페르소나" ON user_personas
  FOR ALL USING (auth.uid() = user_id);

-- persona_items: 본인 페르소나의 항목만
ALTER TABLE persona_items ENABLE ROW LEVEL SECURITY;
CREATE POLICY "사용자 본인 항목" ON persona_items
  FOR ALL USING (persona_id IN (SELECT id FROM user_personas WHERE user_id = auth.uid()));

-- persona_feedback: 본인 페르소나의 피드백만
ALTER TABLE persona_feedback ENABLE ROW LEVEL SECURITY;
CREATE POLICY "사용자 본인 피드백" ON persona_feedback
  FOR ALL USING (persona_id IN (SELECT id FROM user_personas WHERE user_id = auth.uid()));
```

## W5-02: 크롤링 API Route (`/api/persona/crawl`)

### 엔드포인트

```
POST /api/persona/crawl
Body: { blogUrl: string }  // "https://blog.naver.com/youyoubear0517"
Response: { personaId: string, status: "crawling" }
```

### 크롤링 파이프라인

```
1. blogUrl에서 blogId 추출
2. PostTitleListAsync AJAX API로 게시물 logNo 목록 수집 (최대 30편)
3. 각 게시물: m.blog.naver.com/{blogId}/{logNo} fetch
4. cheerio로 파싱:
   a. 텍스트 추출 (제목, 본문, 카테고리)
   b. HTML 메타데이터 추출 (아래 상세)
5. user_personas.crawl_status → 'analyzing'
6. AI 분석 호출 (W5-03)
7. persona_items INSERT
8. user_personas.crawl_status → 'done'
```

### HTML 메타데이터 추출 (cheerio)

`.se-main-container` 내부에서:

```typescript
interface HtmlMetadata {
  // 블록 시퀀스 — 글의 뼈대
  block_sequence: { type: "text" | "image" | "map" | "video" | "oglink" | "sticker" }[];

  // 폰트 컨텍스트
  font_classes: Record<string, number>;    // "se-ff-nanumbareunhipi": 66
  size_classes: Record<string, number>;    // "se-fs-fs16": 66

  // 정렬 패턴
  align_left: number;
  align_center: number;
  center_texts: string[];  // 중앙정렬된 텍스트 샘플

  // 볼드 패턴
  bold_texts: string[];    // <b>, <strong> 안의 텍스트

  // 줄 간격 패턴
  empty_text_blocks: number;   // 내용 없는 se-text 블록 수 (빈 줄 역할)
  total_text_blocks: number;   // 전체 se-text 블록 수

  // 색상 사용
  color_spans: { color: string; text: string }[];  // style="color:..." 있는 span
}
```

### 서버리스 고려사항

- 30편 × 모바일 HTML fetch → 시간 소요 → **Vercel Functions 60초 제한**
- 해결: 10편씩 3번에 나눠 fetch + 분석은 별도 호출
- 또는: `crawl_status` 폴링 방식 (crawl 시작 → 백그라운드 진행 → UI 폴링)

## W5-03: AI 분석 (`lib/ai/analyze-persona.ts`)

### 입력

```typescript
interface CrawledPost {
  title: string;
  body: string;           // 텍스트
  category?: string;
  htmlMeta: HtmlMetadata; // HTML 메타데이터
}
```

### 2-pass 분석

**Pass 1: 콘텐츠 분석** (voice, emoji, structure, ending, forbidden, custom)

```
[시스템 프롬프트]
블로그 글쓰기 스타일 분석 전문가.
주어진 블로그 글들의 공통 패턴을 카테고리별로 추출.

카테고리:
- voice: 어미, 말투, 특유의 표현 (빈도 포함)
- emoji: 사용 이모지와 규칙 (빈도, 위치, 금지 이모지)
- structure: 글 구조 패턴 (오프닝~마무리 흐름)
- ending: 마무리 패턴 (CTA, 추천 문구)
- forbidden: 절대 쓰지 않는 표현
- custom: 자기지칭, 독자호칭 등 독특한 패턴

출력: JSON 배열 [{ category, key, value, priority }]
```

- 글 5편씩 청크 → 각 청크 분석 → 중복 병합 (동일 key는 priority 합산)

**Pass 2: 포맷팅 분석** (formatting)

```
[시스템 프롬프트]
블로그 HTML 포맷팅 패턴 분석 전문가.
SmartEditor HTML 메타데이터에서 시각적 패턴을 추출.

분석 항목:
1. 폰트: 주 사용 폰트 + 사이즈 (제목/본문/강조 구분)
2. 줄 간격: 문장 간, 문단 간, 사진 전후 빈 줄 패턴
3. 정렬: 좌측/중앙 사용 패턴 (어떤 맥락에서 중앙정렬?)
4. 볼드: 어떤 텍스트에 볼드 적용? (장소명, 메뉴명, 가격 등)
5. 색상: 텍스트 색상 사용 여부/패턴
6. 블록 배치: 텍스트-사진 교차 패턴

출력: JSON 배열 [{ category: "formatting", key, value, priority }]
```

- 전체 게시물의 HtmlMetadata 통계 집계 → AI에게 전달

### 출력 (persona_items INSERT)

```typescript
interface PersonaItem {
  category: "voice" | "emoji" | "structure" | "ending" | "forbidden" | "custom" | "formatting";
  key: string;
  value: string;
  priority: number;
  source: "ai";
}
```

예상 항목 수: 콘텐츠 25~40개 + 포맷팅 8~15개 = **총 35~55개**

```typescript
async function handleCopyHtml() {
  if (!draft?.html) return;

  // ClipboardItem으로 리치텍스트 복사
  const blob = new Blob([draft.html], { type: "text/html" });
  await navigator.clipboard.write([
    new ClipboardItem({ "text/html": blob })
  ]);
}
```
