## W1-01: Supabase 프로젝트 생성 + .env.local 설정

**목표**: Supabase 프로젝트를 웹 앱에 연결

**작업 내용**:
1. Supabase 대시보드에서 새 프로젝트 생성 (또는 기존 프로젝트 확인)
2. Project URL, Anon Key, Service Role Key 획득
3. `apps/web/.env.local` 생성:
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
   SUPABASE_SERVICE_ROLE_KEY=eyJ...
   NEXTAUTH_URL=http://localhost:3000
   NEXTAUTH_SECRET=(랜덤 생성)
   ANTHROPIC_API_KEY=(기존 키 또는 신규)
   ```
4. `.env.example` 업데이트 (ANTHROPIC_API_KEY 추가)

**의존성**: 없음 (첫 번째 작업)
**완료 기준**: `npm run dev` 실행 후 Supabase 연결 에러 없음

## W1-02: DB 마이그레이션 실행

**목표**: Supabase에 테이블 생성

**작업 내용**:
1. 기존 마이그레이션 파일 검토 (이미 작성됨):
   - `supabase/migrations/00001_create_users.sql` — users + user_credentials
   - `supabase/migrations/00002_create_generation_queue.sql` — generation_queue + view
2. Supabase SQL Editor에서 마이그레이션 실행
3. RLS (Row Level Security) 정책 추가:
   - users: 본인 레코드만 조회/수정
   - generation_queue: 본인 작업만 조회/생성/수정
   - user_credentials: 본인만 조회/수정
4. 빈 중복 마이그레이션 제거 (`20260306003927_create_users_table.sql`)

**의존성**: W1-01
**완료 기준**: Supabase 대시보드에서 테이블 3개 + RLS 정책 확인

## W1-03: Supabase Storage 버킷 생성

**목표**: 사진 업로드용 Storage 버킷

**작업 내용**:
1. `photos` 버킷 생성 (private)
2. Storage 정책:
   - 인증 사용자만 업로드 가능
   - 본인 폴더(`{user_id}/`)만 접근
   - 파일 크기 제한: 10MB
   - 허용 타입: image/jpeg, image/png, image/webp
3. 업로드 경로 규칙: `photos/{user_id}/{timestamp}_{filename}`

**의존성**: W1-01
**완료 기준**: 테스트 이미지 업로드/다운로드 성공

## W1-04: lib/supabase.ts 클라이언트 설정

**목표**: Supabase 클라이언트 + 타입 정의

**작업 내용**:
1. `lib/supabase.ts` — 브라우저용 클라이언트 (anon key)
2. `lib/supabase-server.ts` — 서버용 클라이언트 (service role key)
3. `lib/database.types.ts` — Supabase CLI로 타입 자동 생성
   ```bash
   npx supabase gen types typescript --project-id <id> > lib/database.types.ts
   ```

**의존성**: W1-01, W1-02
**완료 기준**: 클라이언트에서 users 테이블 조회 성공
