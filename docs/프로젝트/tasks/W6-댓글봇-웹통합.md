# W6: 댓글 봇 웹 통합 — 텔레그램 + 웹 이중 제어

> 댓글 봇의 제어 평면(승인, 설정, 상태)을 Supabase로 이관하여
> 텔레그램과 웹 대시보드 양쪽에서 동일하게 관리할 수 있게 한다.
> 설계 확정: 2026-03-07

---

## 개요

### 현재 문제

```
텔레그램 봇 ──▶ SQLite (로컬)     ← 제어 가능
웹 플랫폼   ──▶ Supabase (클라우드) ← 댓글 봇 접근 불가
```

텔레그램과 웹이 서로 다른 DB를 바라보고 있어서, 웹에서 댓글 봇을 제어할 수 없다.

### 목표 구조

```
┌──────────────┐     ┌──────────────┐
│  텔레그램 봇  │     │  웹 대시보드  │
│  (Python)    │     │  (Next.js)   │
└──────┬───────┘     └──────┬───────┘
       │                    │
       ▼                    ▼
  ┌─────────────────────────────────┐
  │      Supabase (제어 평면)       │
  │  pending_comments               │
  │  bot_settings                   │
  │  bot_run_log                    │
  └──────────────┬──────────────────┘
                 │
                 ▼
  ┌─────────────────────────────────┐
  │    댓글 봇 (Python + Cron)      │
  │    SQLite (운영 데이터)         │
  │    comment_history              │
  │    blogger_visit_log            │
  │    blogger_behavior             │
  └─────────────────────────────────┘
```

### 핵심 결정 사항

| 항목 | 결정 |
|------|------|
| 공유 DB | Supabase (제어 평면만) |
| 로컬 DB | SQLite 유지 (운영 데이터 — 속도·독립성) |
| 봇 인터넷 의존 | 허용 (네이버 접속 자체가 인터넷 필수이므로 추가 리스크 없음) |
| 기존 supabase_client.py | 확장 (get_supabase + get_admin_user_id 재사용) |
| 웹 페이지 | `/bot` — 상태·승인·설정 통합 |
| 텔레그램 봇 | 기존 명령어 유지, DB만 Supabase로 전환 |

## 데이터 분리 원칙

### Supabase로 이관 (제어 평면 — 웹·텔레그램 공유)

| 테이블 | 용도 | 비고 |
|--------|------|------|
| `pending_comments` | 승인 대기 댓글 | 웹·텔레그램 양쪽에서 승인/거부 |
| `bot_settings` | 봇 설정 | 시간대, 한도, auto/manual 모드 |
| `bot_run_log` | 실행 이력 요약 | 웹 대시보드 상태 표시용 |

### SQLite 유지 (운영 데이터 — 봇 전용)

| 테이블 | 용도 | 이유 |
|--------|------|------|
| `comment_history` | 댓글 이력 (UNIQUE 체크) | post_url 중복 체크 속도 (로컬 0ms vs 네트워크 ~100ms) |
| `blogger_visit_log` | 일별 방문 한도 | 실행 중 수십 회 조회 — 로컬 필수 |
| `blogger_behavior` | 오토 블로거 추적 | n-gram 분석 등 빈번한 R/W |
| `retry_queue` | 재시도 큐 | 봇 내부 로직 전용 |

## 데이터 모델 (Supabase 신규/변경)

### pending_comments (신규)

```sql
CREATE TABLE pending_comments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  blog_id TEXT NOT NULL,
  post_url TEXT NOT NULL,
  post_title TEXT NOT NULL DEFAULT '',
  comment_text TEXT NOT NULL,
  ai_generated BOOLEAN NOT NULL DEFAULT true,
  status TEXT NOT NULL DEFAULT 'pending'
    CHECK (status IN ('pending', 'approved', 'rejected', 'posted', 'failed')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  decided_at TIMESTAMPTZ,
  decided_by TEXT,          -- 'web' 또는 'telegram'
  posted_at TIMESTAMPTZ,
  fail_reason TEXT
);

CREATE INDEX idx_pending_status ON pending_comments(status);
CREATE INDEX idx_pending_user ON pending_comments(user_id);

-- RLS: 본인 데이터만 접근
ALTER TABLE pending_comments ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can view own pending comments"
  ON pending_comments FOR SELECT
  USING (auth.uid() = user_id);
-- service_role은 RLS 우회 (봇·텔레그램용)
```

### bot_settings (신규)

```sql
CREATE TABLE bot_settings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL UNIQUE REFERENCES auth.users(id) ON DELETE CASCADE,
  approval_mode TEXT NOT NULL DEFAULT 'manual'
    CHECK (approval_mode IN ('manual', 'auto')),
  weekday_hours JSONB NOT NULL DEFAULT '{"start": 20, "end": 24}',
  weekend_hours JSONB NOT NULL DEFAULT '{"start": 13, "end": 18}',
  max_comments_per_day INT NOT NULL DEFAULT 30,
  max_bloggers_per_day INT NOT NULL DEFAULT 10,
  is_active BOOLEAN NOT NULL DEFAULT true,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE bot_settings ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can manage own bot settings"
  ON bot_settings FOR ALL
  USING (auth.uid() = user_id);
```

### bot_run_log (신규)

```sql
CREATE TABLE bot_run_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  run_at TIMESTAMPTZ DEFAULT NOW(),
  bloggers_visited INT NOT NULL DEFAULT 0,
  comments_written INT NOT NULL DEFAULT 0,
  comments_failed INT NOT NULL DEFAULT 0,
  pending_count INT NOT NULL DEFAULT 0,
  error_message TEXT,
  duration_seconds INT
);

CREATE INDEX idx_run_log_user ON bot_run_log(user_id, run_at DESC);

ALTER TABLE bot_run_log ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can view own run logs"
  ON bot_run_log FOR SELECT
  USING (auth.uid() = user_id);
```

## 기술 참고

### 봇 ↔ Supabase 통신

- Python 봇: `supabase-py` (이미 설치됨) + `service_role` 키 (RLS 우회)
- 웹: `@supabase/supabase-js` (이미 설치됨) + `anon` 키 + RLS

### 실시간 반영 (향후 확장 가능)

- Supabase Realtime 구독으로 웹 UI 자동 갱신 가능
- 현재는 polling (페이지 로드 시 조회)으로 충분
- 필요 시 `supabase.channel('pending_comments').on('INSERT', ...)` 추가

### 마이그레이션 전략

- SQLite `pending_comments`에 기존 데이터가 있으면 1회성 마이그레이션 스크립트 실행
- 이후 SQLite pending 테이블은 사용하지 않음 (bot_settings도 동일)
