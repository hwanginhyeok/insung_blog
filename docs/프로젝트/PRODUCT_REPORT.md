## 1. 프로덕트 개요

### 핵심 기능 3가지

1. **AI 댓글 봇** — 내 블로그에 댓글 남긴 사람들의 블로그에 방문하여 AI가 맥락에 맞는 댓글을 자동 생성. Ollama(로컬) 우선, Anthropic Haiku 폴백. 페르소나 기반 톤 커스터마이징.
2. **게시물 자동 발행** — 사진+메모를 보내면 Claude Vision으로 초안 생성 → 사용자 리뷰 → 네이버 블로그에 자동 게시 (임시저장/발행 모두 지원).
3. **대댓글 자동 답글** — 내 게시물에 달린 방문자 댓글을 수집하여 블로그 주인 톤으로 AI 답글 생성. 페르소나 3분할 (글쓰기/댓글/대댓글).

### 기술 스택

| 영역 | 기술 |
|------|------|
| 백엔드 (봇) | Python 3.12+, FastAPI (uvicorn), asyncio |
| 브라우저 자동화 | Playwright (headless Chromium) |
| AI 모델 | Ollama gemma3:4b (로컬), Anthropic Claude Haiku 4.5 (API) |
| DB (제어 평면) | Supabase (PostgreSQL + RLS + Edge Functions) |
| DB (로컬 운영) | SQLite (유저별 파일 분리: `data/{user_id[:8]}/comments.db`) |
| 웹 프론트엔드 | Next.js 14, TypeScript, Tailwind, shadcn/ui |
| 웹 배포 | Vercel |
| 메시징 | 텔레그램 봇 (python-telegram-bot) |
| 인프라 | WSL2 Ubuntu, systemd --user |

### 현재 아키텍처 (서비스 구조)

```
systemd --user 서비스 (3개)
├── blog-api        — FastAPI 서버 (포트 8001)
├── blog-worker     — 명령 큐 워커 (Supabase 폴링 10초)
└── blog-telegram   — 텔레그램 봇

웹 플랫폼 (Vercel)
├── Next.js 14 (대시보드, AI 글쓰기, 봇 관리)
├── Supabase Auth (카카오/네이버 소셜 로그인)
└── API Routes → Supabase → bot_commands → 워커가 실행

Supabase (공유 제어 평면)
├── bot_commands     — 명령 큐 (웹/텔레그램 → 워커)
├── pending_comments — 댓글 승인 관리
├── bot_settings     — 봇 설정 (시간대, 한도, 모드)
├── bot_run_log      — 실행 이력
├── bot_cookies      — 네이버 쿠키 (AES 암호화)
├── incoming_comments — 대댓글 대상 수집
├── user_personas    — 페르소나 정의
├── persona_items    — 페르소나 항목 (voice, emoji, ending 등)
└── generation_queue — 게시물 생성 큐
```

## 2. 핵심 기능별 현황

### 2.1 댓글 봇

**동작 방식:**
1. 내 블로그 댓글 관리 페이지에서 최근 댓글 작성자(블로거) 수집
2. 각 블로거 방문 → 게시물 3~5개씩 배치로 본문 추출
3. AI 댓글 생성 (Ollama 우선 → Anthropic 폴백 → phrases 정적 폴백)
4. 승인 모드에 따라 즉시 게시(auto) 또는 승인 대기(manual)
5. SQLite + Supabase 이중 기록

**AI 댓글 품질 시스템:**
- 카테고리 자동 감지 (맛집/여행/일상/IT/뷰티/육아/재테크/리뷰)
- 톤 랜덤화 (감탄형/공감형/정보감사형)
- 시작어 중복 방지 (최근 5개 댓글 첫 단어 추출)
- 유사도 체크 (SequenceMatcher 0.7 이상이면 재생성)
- 후처리 파이프라인 (post_processor)
- 오토 블로거 감지 (점수 기반 스킵)

**성능 벤치마크:**

| 모드 | 3개 소요시간 | 분당 처리량 | 비용 |
|------|-------------|-----------|------|
| Ollama 순차 (gemma3:4b) | 7.1초 | 25개/분 | 무료 |
| Ollama 병렬 (PARALLEL=3) | 5.4초 | 33개/분 | 무료 |
| Anthropic Haiku 단건 | ~0.5초 | ~120개/분 | ~₩6,000/월 (100개/일) |

**일일 한도 (기본값):**
- 블로거 10명/일, 댓글 30개/일
- 허용 시간대: 평일 20:00~24:00, 주말 13:00~18:00
- 댓글 간 딜레이: 5~15초, 블로거 간: 30~60초

### 2.2 게시물 자동 발행

**동작 방식:**
1. 웹/텔레그램에서 사진+메모+카테고리 입력
2. Claude Haiku 4.5 Vision으로 이미지 분석 + 초안 생성
3. 카테고리별 스타일 가이드 적용 (맛집/체험일상/재테크/리뷰/개인이벤트)
4. 페르소나 기반 톤 커스터마이징
5. 사용자 리뷰 후 네이버 블로그에 임시저장 또는 직접 발행

**제약사항:**
- 이미지 최대 10장, 20MB/장
- 본문 300~1,500자
- 해시태그 15~25개 자동 생성
- 네이버 에디터 셀렉터 변경 시 수동 업데이트 필요 (debug_publisher.py)
- Playwright 기반이므로 네이버 UI 변경에 민감

### 2.3 대댓글 자동 답글

**동작 방식:**
1. 내 블로그 최근 게시물의 댓글을 수집 (incoming_comments)
2. 기존 답글 여부 확인 (reply_status 체크)
3. 블로그 주인 톤으로 AI 답글 생성
4. 페르소나 3분할: 글쓰기용 / 댓글용(닉네임 항목 제거) / 대댓글용(닉네임 유지)

**한도:**
- 1회 실행 최대 20개 답글
- 하루 최대 50개 답글
- 최근 게시물 10개 스캔

### 2.4 텔레그램 봇 + 웹 대시보드

**텔레그램 봇 기능:**
- 봇 실행/중지, 댓글 승인/거부
- 진행 상황 중간 보고 (30개마다)
- 연속 실패 조기 경고 (3회)
- 쿠키 만료 알림
- 이웃 발견/방문/신청

**웹 대시보드 (apps/web/bot):**
- 3단계 Stepper UI: 봇 실행 → 댓글 리뷰/승인 → 게시
- 실시간 진행 표시 (5건마다 폴링 업데이트)
- 댓글 통계 카드, 분석 차트, 활동 캘린더
- 쿠키 업로드 (AES 암호화 저장)
- 봇 설정 패널 (시간대, 한도, 승인 모드)
- 실행 이력 패널

**제약사항:**
- 웹 → 워커 통신은 Supabase 폴링 (10초 간격) — WebSocket 미구현
- 쿠키 기반 인증 (네이버 2FA 대응 한계)

## 3. 스케일링 분석

### 3.1 10명 (현재~단기)

**현재 인프라로 가능한지:** 가능하지만 빠듯함

**처리 시간 추정:**
- 1명당 봇 실행: 블로거 10명 × 3~5댓글 × (방문 5초 + AI 2초 + 작성 10초) + 딜레이 ≈ 30~40분
- 동시 브라우저: MAX_CONCURRENT_BROWSERS=3 (Elastic Semaphore로 유저별 공정 분배)
- 10명 전체: 유저당 최소 1슬롯 할당 → 3명씩 병렬 → 약 4라운드 × 40분 ≈ 160분 (2시간 40분)

**병목 지점:**
- WSL2 메모리 (Playwright 세션당 ~300~500MB)
- Supabase 무료 티어 커넥션 한도 (동시 60개)
- 허용 시간대(20~24시)에 10명 처리 가능하나 여유 없음

**필요한 변경:**
- 유저별 실행 시간 분산 (시간대 세분화)
- MAX_CONCURRENT_BROWSERS 4로 증가 (RAM 16GB 필요)
- 일일 봇 한도(Freemium Gate) 이미 구현됨 (check_daily_bot_limit RPC)

### 3.2 100명 (중기)

**인프라 요구사항:**
- 전용 서버 (VPS 또는 클라우드) — WSL2 탈출 필수
- RAM 32GB+ (동시 브라우저 8~10개)
- 워커 2~3인스턴스 (worker_id 필드 추가)

**비용 예측:**
| 항목 | 월 비용 |
|------|--------|
| VPS (4Core/32GB) | ₩50,000~80,000 |
| Supabase Pro | $25 (₩35,000) |
| Anthropic API (100명 × 30댓글/일 × 30일) | ₩180,000 (Haiku 기준) |
| Vercel Pro | $20 (₩28,000) |
| **합계** | **₩293,000~323,000** |

**필요한 아키텍처 변경:**
- 다중 워커 분산 실행 (bot_commands에 worker_id 추가)
- Redis/RabbitMQ 기반 큐 (Supabase 폴링 → 이벤트 드리븐)
- 유저별 쿠키 세션 풀 관리
- 시간대별 실행 스케줄러 (09:00~23:00 분산)
- SQLite → PostgreSQL 마이그레이션 (로컬 운영 DB)

### 3.3 1000명 (장기)

**인프라 요구사항:**
- Kubernetes 클러스터 또는 서버리스 아키텍처
- 워커 노드 10~20개 (Auto-scaling)
- RAM 총 256GB+ 분산
- CDN + 로드밸런서

**비용 예측:**
| 항목 | 월 비용 |
|------|--------|
| 클라우드 인프라 (AWS/GCP) | ₩500,000~1,000,000 |
| Supabase Pro + 추가 연결 | ₩100,000 |
| Anthropic API (1000명 × 30댓글/일) | ₩1,800,000 |
| 모니터링 (Datadog/Sentry) | ₩100,000 |
| **합계** | **₩2,500,000~3,000,000** |

**필요한 아키텍처 변경:**
- 메시지 큐 (RabbitMQ/SQS) + 워커 풀
- 브라우저 팜 (Browserless.io 또는 자체 Playwright 클러스터)
- 쿠키/세션 Redis 분산 캐시
- 유저별 격리 (네임스페이스 또는 컨테이너)
- 셀렉터 모니터링 + 자동 복구 시스템
- 요금제별 우선순위 큐 (유료 > 무료)

## 6. 알려진 문제점

### 현재 버그/제약사항

| 항목 | 심각도 | 설명 |
|------|--------|------|
| 네이버 셀렉터 깨짐 | 높음 | 네이버 블로그 UI 업데이트 시 댓글 작성/발행 전량 실패. 수동 확인 필요. |
| 쿠키 만료 | 중간 | NID_AUT 쿠키 만료 시 사용자가 수동으로 재업로드해야 함. 자동 갱신 불가. |
| WSL2 메모리 제한 | 중간 | 동시 브라우저 3개 기준 RAM 6~8GB 필요. 다중 사용자 시 병목. |
| 폴링 지연 | 낮음 | 워커 10초 + 웹 폴링 → 최대 15~20초 UI 반영 지연. |

### 기술 부채

| 항목 | 설명 |
|------|------|
| SQLite 동시 접근 | 다중 사용자 시 WAL 모드로 대응 중이나, 100명+ 시 PostgreSQL 전환 필요. |
| Supabase 폴링 | bot_commands 10초 폴링 → Supabase Realtime 또는 Redis 큐로 전환 필요. |
| 단일 워커 | command_worker.py 1인스턴스. 장애 시 전체 서비스 중단. |
| 하드코딩된 딜레이 | 댓글 간/블로거 간 딜레이가 settings.py에 고정. 유저별 커스터마이징 불가. |
| 레거시 admin 모드 | user_id=None 시 .env 기반 admin 폴백. 다중 사용자 완전 전환 후 제거 필요. |
| 페르소나 캐싱 | 매 실행마다 Supabase 조회. 짧은 TTL 캐싱으로 DB 부하 절감 가능. |
