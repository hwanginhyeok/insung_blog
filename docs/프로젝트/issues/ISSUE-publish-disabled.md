# ISSUE: 네이버 자동 발행/임시저장 기능 비활성화

**날짜**: 2026-04-06
**상태**: 점검 중 (UI 제거, 백엔드 유지)
**영향**: 글쓰기 페이지 (/write)

## 내용

네이버 자동 발행/임시저장 버튼을 UI에서 제거함.
사용자는 "HTML 복사" 버튼으로 텍스트+서식을 복사한 뒤 네이버 에디터에 직접 붙여넣기로 발행.

## 이유

1. HTML 복사 시 base64 이미지 데이터가 포함되면 5MB 네이버 붙여넣기 제한 초과
2. 자동 발행 기능은 셀렉터 깨짐 위험, 재시도 없음, 타임아웃 없음 등 불안정 요소 존재
3. 임시저장도 사용자가 직접 복사+붙여넣기로 충분

## 변경 사항

- `write/page.tsx`: "네이버 발행", "네이버 임시저장" 버튼 제거
- `write/page.tsx`: HTML 복사에서 base64 이미지 변환 제거 (빈 플레이스홀더)
- 백엔드 코드 유지 (`src/publisher/blog_publisher.py`, `command_worker.py`의 handle_publish/handle_save_draft)

## 현재 워크플로우

```
AI 초안 생성 → HTML 복사 (이미지 제외) → 네이버 에디터에 붙여넣기 → 사진 수동 업로드 → 발행
```

## 복구 조건

자동 발행/임시저장 기능을 다시 활성화하려면:
- Playwright 셀렉터 최신화 (`debug_publisher.py`로 확인)
- 타임아웃 + 재시도 로직 추가
- E2E 테스트 통과 확인
