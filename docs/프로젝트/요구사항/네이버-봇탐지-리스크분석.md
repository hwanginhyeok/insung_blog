## 1. 핵심 결론

### 댓글 받는 블로거는 피해 없음
봇 댓글을 받은 블로그가 저품질 되거나 제재받은 사례는 **발견되지 않음**.
네이버는 댓글을 **쓴 계정**을 제재하지, 댓글을 **받은 블로그**를 제재하지 않음.

### Playwright 탐지에 걸리는가?
**안 걸림.** 우리 stealth 설정이 주요 탐지 포인트를 전부 우회함.

## 2. Playwright 탐지 대응 현황

### 네이버가 체크하는 것 vs 우리 대응

| 탐지 시그널 | 네이버 체크 | 우리 대응 | 상태 |
|-----------|-----------|---------|------|
| `navigator.webdriver` | true면 봇 | `--disable-blink-features=AutomationControlled` → false | ✅ 대응 |
| `window.chrome` | 없으면 가짜 | Chromium 기반이라 자동 존재 | ✅ 대응 |
| `navigator.plugins.length` | 0이면 headless | Chromium 기본 플러그인 로드됨 | ✅ 대응 |
| User-Agent | 구버전/비정상 | Chrome 133 최신 UA 설정 | ✅ 대응 |
| Viewport | 비정상 크기 | 1280x800 (실제 모니터) | ✅ 대응 |
| 타이핑 속도 | 즉시 입력 | 0.05-0.15초/글자 시뮬레이션 | ✅ 대응 |
| 행동 패턴 | 기계적 반복 | 정규분포 대기(20-40초) + 랜덤 오프셋 | ✅ 대응 |
| 댓글 내용 | 템플릿 반복 | AI 고유 생성 + 페르소나 | ✅ 대응 |

### 설정 파일: `src/utils/browser.py`

```python
_BROWSER_ARGS = [
    "--no-sandbox",
    "--disable-dev-shm-usage",
    "--disable-blink-features=AutomationControlled",  # navigator.webdriver 숨김
]
```

### 네이버 입장에서 구분 불가

```
일반 유저: Chrome 접속 → 블로그 방문 → 댓글 입력 → 제출
우리 봇:   Chrome 접속 → 블로그 방문 → 댓글 입력 → 제출
(기술적으로 동일)
```

## 3. 2025년 네이버 탐지 강화 내용

- 2025년 7월부터 Selenium/봇 기반 접근 차단 강화
- 대상: `navigator.webdriver=true`인 봇, WebDriver 미은닉 매크로
- 2024 하반기 제재 게시물의 95%가 AI 스팸, 91.2%를 AI 시스템이 자동 탐지
- **우리 수준의 stealth 처리된 Playwright는 해당 없음**

## 4. 걸리는 시나리오 (기술이 아닌 행동 패턴)

| 시나리오 | 위험도 | 우리 상황 |
|---------|--------|---------|
| 같은 IP에서 10개+ 계정 활동 | 높음 | 동글/프록시로 대응 예정 (20명+) |
| 하루 400개+ 댓글 | 높음 | 30개 제한 — 안전 |
| 매일 정확히 같은 시간+패턴 | 중간 | 랜덤 오프셋 + 정규분포 대기 적용 |
| 댓글 내용 반복/템플릿 | 중간 | AI 고유 생성 — 안전 |
| 단시간 대량 활동 | 중간 | 20-40초 간격 — 안전 |

## 8. 종합 리스크 평가

| 리스크 | 수준 | 비고 |
|--------|------|------|
| Playwright 기술 탐지 | **낮음** | stealth 설정으로 대응 완료 |
| IP 기반 다수 계정 탐지 | 중간 → 낮음 | 동글/프록시로 대응 예정 |
| 봇 사용 계정 정지 | 중간 | 하루 30개는 정상 범위이나 0%는 아님 |
| 이웃 블로거 피해 | **매우 낮음** | 제재 사례 없음 |
| 법적 처벌 | **매우 낮음** | 대법원 무죄 선례 |

## 9. 첫 고객 안내 사항

1. **이웃 블로거 피해 없음** — 댓글 받는 쪽은 제재 대상 아님
2. **본인 계정 리스크 존재** — 최악의 경우 30일 정지 (확률 낮음)
3. **수동 승인 모드 필수** — 첫 2주는 모든 댓글 직접 확인
4. **하루 10개부터 시작** — 점진적 증가 권장

## 10. 모니터링 & 버그 리포팅

### 감시 항목

| 항목 | 감시 방법 | 대응 |
|------|---------|------|
| 댓글 작성 실패율 급증 | 워커 로그 + 텔레그램 알림 | 네이버 정책 변경 확인 → 셀렉터/stealth 업데이트 |
| 로그인 실패 (캡챠 발생) | 쿠키 검증 로직 | 쿠키 재업로드 안내 |
| 계정 정지 알림 | 사용자 직접 리포트 | 원인 분석 → 대응 문서 업데이트 |
| 네이버 stealth 탐지 변경 | `navigator.webdriver` 체크 주기적 검증 | browser.py args 업데이트 |

### 문제 발생 시 버그 리포팅

- 웹 대시보드: FeedbackPanel (우하단 물음표 버튼)
- 텔레그램: 관리자 채널 자동 알림
- 로그 확인: `journalctl --user -u blog-worker -n 50 --no-pager`
