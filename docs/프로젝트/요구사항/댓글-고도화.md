# 기능 2: 댓글 품질 고도화

> 상태: 요구사항 정의 완료, 일부 구현 (길이 균일화)

## 목적

AI 댓글의 품질을 높여서 자연스럽고 일관된 댓글을 생성하기

## 현재 문제점 (AS-IS)

### 2-1. 댓글 길이 불균일 (수정 완료)
- 프롬프트에 100~200자 지정했지만 실제 23~257자로 편차 큼
- 20~40자 한줄 댓글이 45% 차지
- **수정 완료**: 최소 80자 강제 + 프롬프트 "120~180자, 한줄 금지"

### 2-2. 댓글 패턴 반복
- 같은 블로거의 여러 게시물에 비슷한 구조의 댓글
- "~네요!", "~좋아요!", "감사합니다!" 패턴 반복
- 중복 체크(SequenceMatcher 0.7)가 있지만 구조적 유사성은 못 잡음

### 2-3. 본문 미로드 시 폴백 품질
- 본문이 20자 미만이면 phrases.py 고정 문구 폴백
- 고정 문구가 너무 뻔해서 AI 티가 남

### 2-4. 카테고리별 댓글 스타일 미분화
- 맛집, 여행, 일상, 육아, IT 등 카테고리에 따라 톤이 달라야 함
- 현재는 일률적인 "30대 직장인" 페르소나

## 개선 요구사항 (TO-BE)

### 1. 댓글 다양성 강화

- **톤 랜덤화**: 호출 시마다 톤을 랜덤 선택
  - 감탄형: "와 이건 진짜..."
  - 공감형: "저도 이거 겪어봤는데..."
  - 질문형(가벼운): "혹시 주차는 편했어요?"는 아니지만 "다음에 갈 때 참고할게요" 식
  - 정보 감사형: "이 정보 진짜 유용하네요"
- **시작 문구 금지 목록**: "와 ", "오 ", "진짜 " 등 반복 시작어를 금지 목록 관리
- **최근 5개 댓글의 시작 문구 중복 방지**: 같은 단어로 시작하지 않도록

### 2. 카테고리 감지 + 스타일 분화

- 게시물 본문/제목에서 카테고리 자동 감지:
  - 맛집/카페 → 메뉴, 가격, 분위기 중심
  - 여행 → 장소, 풍경, 팁 중심
  - 일상/육아 → 공감, 응원 중심
  - IT/리뷰 → 스펙, 사용감 중심
- AI 프롬프트에 카테고리 힌트 전달

### 3. phrases.py 폴백 개선

- 고정 문구를 더 자연스럽게 업데이트
- 카테고리별 폴백 문구 분리
- 최소 20개 이상의 다양한 폴백 확보

### 4. 댓글 후처리

- 이모지 과다 사용 방지 (최대 2개)
- "ㅎㅎ", "ㅋㅋ" 중복 사용 방지 (1개만)
- 마침표 통일 (문장 끝 스타일 일관성)

## 구현 대상 파일

| 파일 | 작업 |
|------|------|
| `src/commenter/ai_comment.py` | 톤 랜덤화, 시작어 중복 방지, 카테고리 힌트 |
| `src/commenter/phrases.py` | 폴백 문구 다양화 + 카테고리별 분리 |
| `src/commenter/comment_post_processor.py` | 신규 — 이모지/ㅎㅎ 정리 등 후처리 |

## 우선순위

1. 톤 랜덤화 + 시작어 중복 방지 (효과 큼, 구현 쉬움)
2. 카테고리 감지 + 스타일 분화 (효과 큼, 구현 중간)
3. 후처리 로직 (효과 중간, 구현 쉬움)
4. phrases 폴백 개선 (효과 낮음, 구현 쉬움)
