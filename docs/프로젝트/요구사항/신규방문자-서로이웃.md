# 기능 1: 신규 방문자 댓글 + 서로이웃 신청 + 이웃 발견/방문 자동화

> 상태: 핵심 구현 완료 (2026-03-21)

## 목적

1. 내 블로그에 댓글을 남긴 **신규 방문자**에게 답방 댓글 + 서로이웃 신청
2. **새로운 블로거 발견**: 키워드/테마 기반으로 네이버 검색하여 새 블로거 탐색
3. **기존 이웃 재방문**: 최근 방문하지 않은 이웃을 자동 방문하여 AI 댓글 작성

## 흐름 요약

### A. 기존 답방 흐름 (AS-IS → TO-BE)
```
내 블로그 댓글 작성자 수집
  → 이웃 여부 확인 (이웃 O → 기존 답방만)
  → 이웃 여부 확인 (이웃 X → 답방 댓글 + 서로이웃 신청)
```

### B. 이웃 발견 흐름 (신규)
```
웹 UI "이웃 찾기" 버튼 (키워드 입력 또는 등록된 테마 사용)
  → bot_commands INSERT (discover_neighbors)
  → 워커: 네이버 블로그 검색 → blog_id 추출
  → neighbors 테이블에 type='discovered'로 저장
  → 웹 UI에 발견된 블로거 목록 표시
```

### C. 이웃 방문 흐름 (신규)
```
웹 UI "이웃 방문" 버튼
  → bot_commands INSERT (visit_neighbors)
  → 워커: neighbors 테이블에서 last_interaction_at 오래된 순 선택 (7일 이상)
  → 각 이웃 블로그 방문 → 게시물 수집 → AI 댓글 배치 생성
  → pending_comments에 승인 대기 저장 (manual 모드)
  → last_interaction_at 갱신
```

## 상세 요구사항

### 1. 이웃 여부 확인 — ✅ 구현 완료

- 블로그 방문 시 프로필 영역에서 이웃 상태 감지
- `src/neighbor/neighbor_checker.py`
- 판별 결과: `mutual` / `one_way_following` / `one_way_follower` / `None`

### 2. 서로이웃 신청 — ✅ 구현 완료

- 웹 UI 신청 폼 → `bot_commands` INSERT → 워커 실행 → Playwright 자동화
- `src/neighbor/neighbor_requester.py`
- 일일 한도: `max_neighbor_requests_per_day` (기본 10)
- 신청 이력: `neighbor_requests` 테이블

### 3. 이웃 발견 (키워드/테마 검색) — ✅ 구현 완료

- 네이버 블로그 검색(`search.naver.com`) 결과에서 blog_id + 닉네임 추출
- `src/neighbor/neighbor_discoverer.py`
- 키워드당 최대 10명, 1회 최대 20명 발견
- 기존 이웃 자동 필터링 (중복 방지)
- `neighbor_type='discovered'`로 구분 저장

### 4. 이웃 방문 (미방문 이웃 자동 방문) — ✅ 구현 완료

- `neighbors` 테이블에서 `last_interaction_at` 기준 7일 이상 지난 이웃 선택
- NULL first → 한 번도 방문하지 않은 이웃 우선
- 1회 최대 5명 방문, 이웃당 최대 3개 게시물
- 기존 AI 댓글 배치 생성 로직 재사용 (`generate_comments_batch`)
- `src/neighbor/neighbor_visitor.py`

### 5. 블로그 테마 등록 — ✅ 구현 완료

- `bot_settings.blog_themes` JSONB 컬럼 (예: `["정치", "경제"]`)
- 웹 UI에서 테마 등록/수정 (인라인 편집)
- "테마로 이웃 찾기" 버튼: 등록된 테마를 키워드로 자동 사용
- `api/neighbor/themes/route.ts` (GET/POST)

### 6. DB/이력 관리 — ✅ 구현 완료

- `neighbors` 테이블: 이웃 목록 + 타입 (mutual/following/follower/discovered)
- `neighbor_requests` 테이블: 서로이웃 신청 이력
- `neighbor_interactions` 테이블: 교류 기록 (댓글/방문)
- `neighbor_recommendations` 테이블: 이웃 추천 (향후)
- Supabase RLS 정책: `auth.uid() = user_id`

### 7. 설정 — ✅ 구현 완료

- `bot_settings` 컬럼:
  - `auto_neighbor_request` BOOLEAN (기본 false)
  - `max_neighbor_requests_per_day` INTEGER (기본 10)
  - `neighbor_request_message` TEXT
  - `blog_themes` JSONB (기본 [])

## 미연동 (향후)

| 항목 | 설명 |

| 추천 알고리즘 | neighbor_recommendations 데이터 생성 로직 |
| 댓글 작성 시 interaction 자동 기록 | interaction_tracker.py → comment_writer 연동 |

## 주의사항

- 네이버 서로이웃 신청 UI는 팝업/모달 형태 → DOM 분석 필수
- 하루 서로이웃 신청 과다 시 차단 위험 → 보수적 한도 (10건/일)
- 이웃 발견/방문도 봇 감지 회피 딜레이 적용 (delay_between_bloggers)
- 비공개 블로그나 서로이웃 신청 불가 블로그 대응 필요
