# 기능 1: 신규 방문자 댓글 + 서로이웃 신청 자동화

> 상태: 요구사항 정의 완료, 구현 대기

## 목적

내 블로그에 댓글을 남긴 사람 중 아직 이웃이 아닌 **신규 방문자**에게:
1. 답방 댓글을 달고
2. 서로이웃 신청까지 자동으로 보내기

## 현재 흐름 (AS-IS)

```
내 블로그 댓글 작성자 수집 → 블로그 방문 → 댓글 작성
```
- 이웃/비이웃 구분 없이 모든 댓글 작성자에게 답방
- 서로이웃 신청 기능 없음

## 목표 흐름 (TO-BE)

```
내 블로그 댓글 작성자 수집
  → 이웃 여부 확인 (이웃 O → 기존 답방만)
  → 이웃 여부 확인 (이웃 X → 답방 댓글 + 서로이웃 신청)
```

## 상세 요구사항

### 1. 이웃 여부 확인

- 블로그 방문 시 이웃 상태를 감지
- 네이버 블로그 프로필 영역에서 "이웃추가" 버튼 존재 여부로 판별
  - "이웃추가" 버튼 있음 → 아직 이웃 아님 (신규)
  - "서로이웃" 또는 이웃 표시 → 이미 이웃
- mainFrame 내에서 셀렉터로 확인

### 2. 서로이웃 신청

- 이웃이 아닌 블로거의 블로그 방문 시 서로이웃 신청
- 신청 흐름:
  1. 블로그 프로필 영역의 "이웃추가" 버튼 클릭
  2. "서로이웃 신청" 옵션 선택
  3. 신청 메시지 입력 (AI 생성 또는 고정 템플릿)
  4. 확인 버튼 클릭
- 신청 메시지: 블로그 내용에 맞는 자연스러운 인사 (AI 생성)
- 일일 한도 설정 필요 (네이버 차단 방지)

### 3. 실행 순서

답방 댓글 작성 → 서로이웃 신청 순서로 진행
(댓글을 먼저 달아야 신청 수락 확률이 높음)

### 4. DB/이력 관리

- `neighbor_request_log` 테이블 추가:
  - blog_id, requested_at, status (sent/accepted/rejected)
  - 이미 신청한 블로거에게 중복 신청 방지
- Supabase에도 동기화 (웹 대시보드 확인용)

### 5. 설정

- bot_settings에 추가:
  - `auto_neighbor_request`: 서로이웃 자동 신청 ON/OFF
  - `max_neighbor_requests_per_day`: 일일 신청 한도 (기본 10)
  - `neighbor_request_message`: 신청 메시지 템플릿 (빈 값이면 AI 생성)

## 구현 대상 파일

| 파일 | 작업 |
|------|------|
| `src/collectors/neighbor_checker.py` | 신규 — 이웃 여부 확인 |
| `src/neighbor/neighbor_requester.py` | 신규 — 서로이웃 신청 자동화 |
| `src/storage/database.py` | neighbor_request_log 테이블 추가 |
| `src/orchestrator.py` | 이웃 체크 + 신청 로직 삽입 |
| `config/settings.py` | 서로이웃 관련 상수 추가 |

## 주의사항

- 네이버 서로이웃 신청 UI는 팝업/모달 형태일 수 있음 → DOM 분석 필수
- 하루 서로이웃 신청 과다 시 차단 위험 → 보수적 한도 (10건/일)
- 비공개 블로그나 서로이웃 신청 불가 블로그 대응 필요
