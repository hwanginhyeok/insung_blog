## ✅ 확정된 사항

### 1. 승인 방식 (파라미터 설정)
- **초기**: 모두 수동 승인 (텔레그램으로 브리핑 → 승인 시 작성)
- **전환**: 충분히 검증 후 자동 승인 모드로 전환 가능
- **결과 보고**: 실행 완료 후 텔레그램으로 리포트 → 다음 실행 모드 선택

### 2. 시간대 설정 (텔레그램 파라미터)
- **평일**: 20:00~24:00 (기본)
- **주말**: 13:00~18:00 (오후 확장)
- **커스텀**: 사용자가 직접 시간대 설정 가능

### 3. 캡차 대응
- 캡차 감지 시 즉시 텔레그램 알림
- 수동 개입 (사용자가 캡차 풀면 재개)
- 또는 해당 게시물 스킵하고 계속 진행 (설정 가능)

### 4. 실패 재시도
- 실패한 게시물을 큐에 저장
- 다음 실행 시 우선적으로 재시도
- 최대 3회 재시도 후 포기

### 5. 문구 생성
- **AI 우선**: Claude Haiku로 생성
- **폰백**: AI 실패 시 phrases 사용
- (이미 구현됨, 유지)

## 🛠️ 구현 필요 사항

### DB 스키마 추가
```sql
-- 설정 테이블
bot_settings (
    key TEXT PRIMARY KEY,
    value TEXT,
    updated_at TIMESTAMP
)

-- 실패 재시도 큐
retry_queue (
    id INTEGER PRIMARY KEY,
    blog_id TEXT,
    post_url TEXT,
    post_title TEXT,
    fail_count INTEGER DEFAULT 0,
    last_try_at TIMESTAMP,
    next_try_at TIMESTAMP
)
```

### 텔레그램 명령어
```
/settings                    - 현재 설정 확인
/set_mode {manual|auto}      - 승인 모드 설정
/set_weekday {start} {end}   - 평일 시간대 설정 (예: 20 24)
/set_weekend {start} {end}   - 주말 시간대 설정 (예: 13 18)
/set_captcha {skip|notify}   - 캡차 대응 방식
/retry_now                   - 실패 큐 즉시 재시도
/clear_retry                 - 재시도 큐 초기화
```
