## First Impression

랜딩 페이지에서 3가지가 잘 됨:
- 헤드라인 "경험에 집중하세요. 글은 AI가 쓸게요." — 명확하고 직관적
- 작동 방식 3단계 — 사진+메모→AI→복사 흐름이 이해 가능
- 전체 레이아웃 — AI slop 3열 카드 패턴 피함, 좌우 비대칭이 의도적

하지만 소비자 시뮬레이션에서 **클릭 2번 만에 막힘** — 아래 참조.

## CRITICAL (P0) — 즉시 수정 필요

### C-01: 카카오 로그인 JS SDK 차단
**현상:** 로그인 페이지 로드 시 콘솔 에러 발생, 카카오 버튼 클릭 무효화.
```
Failed to find a valid digest in the 'integrity' attribute for resource
'https://t1.kakaocdn.net/kakao_js_sdk/2.7.4/kakao.min.js'
The resource has been blocked.
```
**원인:** 카카오 SDK CDN 파일 해시가 변경됐는데 `integrity` 어트리뷰트가 구 버전 해시를 가리킴.
**영향:** 카카오로 로그인 실제 작동 안 함 (주요 소셜 로그인 채널 중 하나).
**수정:** integrity 해시 업데이트 또는 integrity 어트리뷰트 제거.

### C-02: Vercel insights 스크립트 404
**현상:** 개발 환경에서 `http://localhost:3098/_vercel/insights/script.js` 로드 실패.
```
Refused to execute script... MIME type ('text/html') is not executable
```
**원인:** `localhost:3001`인데 3098 포트를 참조함. next.config.mjs에 개발/프로덕션 분기 없음.
**영향:** 에러 로그가 개발 중 노이즈가 됨. 프로덕션에서 analytics 수집 안 될 가능성.

## HIGH IMPACT (P1) — 신규 사용자 이탈

### H-01: "무료로 시작하기" → 로그인 폼 (기본)
**현상:** 랜딩의 CTA 클릭 시 `/login`으로 이동, 로그인 폼이 기본으로 표시됨.
**기대:** "시작하기"를 클릭하면 회원가입 폼이 나와야 함.
**현실:** `mode` 기본값이 `"login"`. 신규 사용자는 "계정이 없으신가요? 회원가입" 작은 링크를 찾아야 함.
**수정:** `?signup=1` 쿼리 파라미터로 mode 초기화, 또는 랜딩 CTA href를 `/login?mode=signup`으로 변경.

### H-02: 로그인 후 기본 화면 = 캘린더
**현상:** 로그인 성공 시 `/calendar`로 리디렉트됨.
```tsx
const redirect = rawRedirect.startsWith("/") ? rawRedirect : "/calendar";
```
**기대:** 신규 사용자 첫 화면은 `/write` (핵심 기능) 또는 온보딩이어야 함.
**현실:** 빈 캘린더가 나옴 → 뭘 해야 할지 모름.
**수정:** 신규 여부 판단 후 첫 방문이면 `/write`, 재방문이면 마지막 페이지 또는 `/calendar`.

### H-03: 온보딩 카피 오류 — "왼쪽 메뉴"
**현상:** 온보딩 3단계에서 "**왼쪽 메뉴에서** '글쓰기'를 눌러 첫 글을 만들어보세요."
**현실:** 네비게이션은 **상단**에 있음. 왼쪽 사이드바 없음.
**영향:** 신규 사용자가 왼쪽을 찾다가 혼란.
**수정:** "상단 메뉴에서 '글쓰기'를 눌러..." 로 변경.

### H-04: 회원가입 후 피드백 없음
**현상:** 이메일 가입 후 별도 안내 없이 로그인 폼으로 전환만 됨.
```tsx
setMode("login");
setIsLoading(false);
return;
```
**기대:** Supabase 기본 설정에서 이메일 인증 필요 → "이메일을 확인하세요." 안내 필요.
**현실:** 인증 메일을 받기 전에 로그인 시도 → 실패 → 사용자는 뭐가 문제인지 모름.
**수정:** 가입 후 `setError(null)` 대신 성공 메시지 표시: "가입 완료! 이메일을 확인해 인증을 완료하세요."

### H-05: 비밀번호 찾기 없음
**현상:** 로그인 폼에 "비밀번호를 잊으셨나요?" 링크 없음.
**영향:** 비밀번호 분실 사용자 = 영구 이탈.
**수정:** Supabase `resetPasswordForEmail()` 연동 + 링크 추가.

### H-06: 로그인 페이지에서 홈으로 돌아가는 링크 없음
**현상:** `/login`에 접근 후 홈(`/`)으로 돌아갈 방법이 없음.
**수정:** "인성이" 타이틀에 `href="/"` 추가, 또는 상단에 "← 홈으로" 링크 추가.

## MEDIUM IMPACT (P2) — UX 마찰

### M-01: 로고 클릭 → /write (홈 기대)
**현상:** 헤더의 "인성이" 로고 클릭 시 `/write`로 이동.
```tsx
<Link href="/write" className="text-lg font-bold text-primary">인성이</Link>
```
**기대:** 로고는 보통 홈(`/`) 또는 대시보드(`/dashboard`)로.
**수정:** `href="/"` 또는 `href="/dashboard"` 로 변경.

### M-02: 글쓰기 드롭다운에 5개 항목 — 정보 과부하
**현상:** 글쓰기 탭 클릭 시 글쓰기/캘린더/내 글/페르소나/성과분석 5개 노출.
**기대:** 처음 오는 사용자가 압도됨. "지금 뭘 해야 하지?"
**수정안:** 주 작업(글쓰기, 내 글)만 상단에, 설정성(페르소나, 성과분석)은 하위로.

### M-03: 헤더에 이메일 풀텍스트 표시
**현상:** `{user.email}` — 긴 이메일(예: very.long.email@example.com)이 헤더 공간 침범.
**수정:** 닉네임 우선, 없으면 이메일을 `@` 앞까지만, 또는 아이콘으로 대체.

### M-04: 모바일 스탯 카드 완전 숨김
**현상:** 히어로 우측 스탯 카드 (`1분`, `30개`, `24/7`) — `hidden md:block`.
**영향:** 모바일 방문자는 핵심 가치 제안 중 일부를 못 봄.
**수정:** 모바일에서는 인라인 배지(badge) 형태로 헤드라인 아래 표시.

### M-05: 삭제 시 window.confirm()
**현상:** 글 삭제 시 `window.confirm()` 브라우저 기본 다이얼로그 사용.
```tsx
if (!window.confirm("이 글을 삭제하시겠습니까? 복구할 수 없습니다.")) return;
```
**수정:** shadcn/ui AlertDialog 또는 인라인 확인 패턴으로 교체.

### M-06: 카카오/네이버 버튼 vs 로그인 버튼 — 색상 충돌
**현상:** 메인 "로그인" 버튼과 "네이버로 로그인" 버튼 둘 다 녹색.
**영향:** 눈에 띄는 두 버튼이 같은 색이라 위계가 없음.
**수정:** 메인 로그인 버튼은 emerald, 소셜 버튼은 브랜드 컬러 유지 (카카오=노란색, 네이버=초록색은 OK — 단 로그인 기본 버튼은 다른 색으로).

## POLISH (P3) — 작은 개선

### P-01: 터치타겟 h=19px
랜딩 두 번째 "무료로 시작하기" `<a>` 링크가 높이 19px.
`<Link>` 안의 텍스트 인라인 요소 패딩 추가 필요.

### P-02: 브랜드 폰트 없음
전체 `ui-sans-serif, system-ui, sans-serif` 만 사용.
AI 블로그 파트너라면 좀 더 독특한 타이포 방향 검토 가치 있음.

### P-03: 랜딩 네비게이션 없음
상단에 네비바가 없음 (로그인 링크조차 CTA 버튼만).
로그인한 사용자가 `/` 로 돌아오면 다시 가입 페이지처럼 보임.

### P-04: 성과분석(Analytics) 페이지 — 실제 데이터 있나?
페이지 존재하는데 어떤 지표를 보여주는지 불명확. 글쓰기 전 방문하면 빈 화면일 가능성.

### 시나리오 A: 처음 오는 네이버 블로거
```
랜딩 → "무료로 시작하기" 클릭
→ 로그인 폼 (기대: 회원가입) [H-01 발동]

→ 이탈 확률 높음
```
**판정: 이탈 위험**

```
랜딩 → 로그인 → "카카오로 로그인" 클릭
→ SDK 차단으로 작동 안 함 [C-01 발동]
→ 버튼 클릭해도 아무 반응 없음
→ 이탈
```
**판정: 기능 불능 (P0)**

### 시나리오 C: 로그인 성공한 사용자 (기존 가입자)
```
로그인 → /calendar 리디렉트 [H-02 발동]
→ 빈 캘린더 화면
→ 온보딩 팝업: "왼쪽 메뉴에서 글쓰기를..." [H-03 발동 — 왼쪽 메뉴 없음]
→ "글쓰기" 드롭다운 클릭 → 5개 항목 노출 [M-02 발동]
→ "글쓰기" 선택 → write 페이지 도착
→ 사진 업로드 + 메모 입력 → AI 생성
→ "HTML 복사" → 네이버 에디터에 붙여넣기 (불편하지만 기능함)
```
**판정: 온보딩 마찰 크지만 핵심 기능 동작**

## CEO 관점 — 제품 방향성 질문

1. **"HTML 복사 → 네이버 발행"이 핵심 마찰 포인트.** 사용자가 직접 네이버 에디터를 열고 붙여넣어야 함. 임시저장 기능이 있다면 왜 그게 기본 흐름이 아닌가? 자동 발행 vs 수동 발행 선택지를 명확히 제시해야 함.

2. **"1분 글 하나 완성"이라는 수치가 검증되지 않음.** 랜딩의 스탯이 실제 데이터인지 추정치인지 불분명. 사용자가 "그렇게 빠르게 안 된다"고 느끼는 순간 신뢰가 깨짐.

3. **타겟 사용자가 "블로거"인데 프로덕트가 "쿠키 등록"을 요구함.** 이 요구사항이 얼마나 많은 잠재 사용자를 이탈시키는지 추적해야 함. 랜딩에 쿠키 언급이 없다가 온보딩에서 처음 나옴 — 놀람 효과.

4. **카카오 로그인이 작동 안 한다면 한국 사용자의 주요 진입 경로가 막힌 것.** 네이버 블로거 타겟이라면 네이버 로그인이 가장 중요한 소셜 로그인 채널일 것. 이 부분 프로덕션 환경에서 검증 필요.
