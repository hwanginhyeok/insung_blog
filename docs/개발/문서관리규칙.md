## 📝 문서 작성 규칙

### Markdown 문법
- **헤더**: `#`부터 `####`까지만 사용 (`#####` 이상 지양)
- **코드 블록**: 언어 지정 필수 (```typescript, ```python 등)
- **다이어그램**: Mermaid 또는 ASCII Art 사용
- **링크**: 상대 경로 사용 (예: `./00-system-overview.md`)

### 버전 관리
- **버전 형식**: `MAJOR.MINOR` (예: 1.0, 1.1, 2.0)
- **MAJOR**: 아키텍처 변경, 하위 호환성 없는 변경
- **MINOR**: 내용 추가, 수정, 하위 호환성 유지하는 변경

### Changelog 형식
```markdown
## Changelog

- **2026-03-05**: 초기 문서 작성 (v1.0)
- **2026-03-07**: API 엔드포인트 추가 (/api/generate) (v1.1)
- **2026-03-10**: DB 스키마 변경 (users 테이블에 phone 추가) (v2.0)
```

## 📁 파일 네이밍 규칙

### 문서 파일
- **숫자 prefix**: `00-`, `01-` 등으로 순서 표시
- **소문자 + 하이픈**: `system-overview.md` (snake_case 지양)
- **언어**: 한국어 파일명 사용 가능

### 코드 파일
- **Web (TypeScript)**: PascalCase (컴포넌트), camelCase (유틸)
- **Worker (Python)**: snake_case
- **SQL**: snake_case, 숫자 prefix (마이그레이션 순서)

## 🚫 금지 사항

1. **민감 정보 문서화**
   - API Key, 비밀번호, 개인정보를 문서에 작성 금지
   - `.env.example`에는 키 이름만, 값은 비워두기

2. **중복 문서**
   - 동일 내용을 여러 문서에 복사붙여넣기 금지
   - 링크로 참조하기

3. **비구조적 변경**
   - 문서 구조(헤더 레벨, 섹션 순서) 무작정 변경 금지
   - 변경 시 관련 문서 모두 확인

4. **미검토 커밋**
   - 문서 수정 후 바로 커밋 금지
   - 최소 한 번 읽어보기
