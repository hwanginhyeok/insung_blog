## 📊 Phase별 확장 전략

| Phase | 사용자 수 | Worker 구성 | 인프라 비용 | 주요 변경사항 |
|-------|-----------|-------------|-------------|---------------|
| **0** | 1명 (형) | Worker-01 (집 PC) | $5/월 | MVP, 단일 사용자 최적화 |
| **1** | 2~5명 | Worker-01 (멀티유저) | $5/월 | 코드만 변경, 동일 PC |
| **2** | 6~20명 | Worker-01 + Worker-02 | $45/월 | VPS 추가, 로드밸런싱 |
| **3** | 20명+ | Worker Pool (3대+) | $100+/월 | 자동 스케일링 고려 |

## ⚠️ 아키텍처 제약사항

1. **Vercel Serverless Timeout**: API Routes 10초 제한 (Hobby 플랜)
   - 해결: 긴 작업은 Queue에 넣고 비동기 처리

2. **Worker 가용성**: 집 PC 재부팅/인터넷 단절 시 중단
   - 해결: 헬스체크 + 자동 재시도 큐

3. **네이버 IP 차단**: 다중 계정 동시 접속 시 리스크
   - 해결: Phase 2부터 프록시 로테이션
