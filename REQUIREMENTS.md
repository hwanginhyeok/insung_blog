# 네이버 블로그 자동화 시스템 요구사항

> 작성일: 2026-03-01
> 프로젝트: 인성이프로젝트 확장판

---

## 1. 시스템 개요

네이버 블로그 운영 전 과정을 자동화하는 시스템.
- **글 작성**: 텔레그램으로 사진+메모 전송 → AI 초안 생성 → 승인 → 네이버 게시
- **댓글 봇**: 내 블로그 댓글 작성자 수집 → 방문 → 본문 읽고 맞춤 댓글 작성
- **AI 업그레이드**: 게시물마다 피드백 수집 → 글쓰기 스킬 개선

---

## 2. 기능 요구사항

### 2-A. 게시물 자동 작성 및 게시

#### 입력
- **채널**: 텔레그램 봇
- **입력 형식**:
  - 사진 1장 이상 (필수)
  - 텍스트 메모 (선택): 장소명, 메뉴, 가격, 느낀 점 등 키워드 수준

#### AI 글 생성 프로세스
1. 사진 → Claude Vision으로 분석 (음식/장소/분위기 파악)
2. 메모 + 사진 분석 결과 → 초안 생성
3. 초안 → 텔레그램으로 반환
4. 사용자 승인 or 수정 요청
5. 승인 시 → 네이버 블로그 자동 게시

#### 글쓰기 스타일
- 주제: 맛집, 여행, 협찬 식당 후기, 일상
- 구체적인 스타일은 피드백 루프를 통해 점진적으로 정의
- 초기 스타일: 친근한 구어체, 사진 설명 + 개인 감상 포함

#### 네이버 게시 방식
- Playwright로 스마트에디터 조작
- 사진 업로드 → 제목 입력 → 본문 입력 → 게시 버튼
- 카테고리: 추후 결정

#### 피드백 루프 (AI 업그레이드 핵심)
- 게시 후 텔레그램으로 피드백 요청
- 피드백 형식: 좋았던 점 / 수정이 필요한 점 / 추가 지시사항
- 피드백 → `skills/writing_style.md` 파일에 누적 저장
- 다음 글 생성 시 해당 파일 자동 참조

### 2-B. 댓글 자동 답방 봇

#### 수집 방식
- 내 블로그에 댓글 달아준 사람 목록 수집
- 기존 코드 유지 + 관리자 페이지 대안 방식 추가

#### 방문 및 댓글 작성
- 방문자 블로그 최근 게시물 방문
- **게시물당 1~3개 랜덤** 댓글 작성
- **댓글 방식 변경**: 고정 문구(phrases.py) → AI 생성 방식
  - 해당 게시물 본문을 읽고 내용에 맞는 호의적 댓글
  - 글의 핵심 내용을 언급하여 진짜 읽은 것처럼 자연스럽게
  - 예: "오늘 소개하신 파스타 정말 맛있어 보여요! 크림소스랑 면의 비율이 딱 좋아 보이던데 직접 먹어보고 싶네요 😊"

#### 실행 시간대
- **오후 8시 ~ 자정 (20:00 ~ 24:00)** — 블로그 접속자 많은 시간대
- 기존 ALLOWED_HOUR_START/END 설정 변경

#### 봇 감지 방지
- 글자 단위 타이핑 (기존 유지)
- 댓글 사이 랜덤 딜레이 (기존 유지)
- AI 생성 댓글로 다양성 확보 (신규)

---

### 2-C. 텔레그램 봇 인터페이스

#### 글 작성 플로우
```
사용자: [사진 1~5장] + "강남 이탈리안 레스토랑, 파스타 2만원대, 분위기 좋음"
봇: 초안 생성 중... (30초~1분)
봇: [초안 전문 전송]
    ────────────────
    ✅ 게시  |  ✏️ 수정  |  🔄 다시쓰기
봇(승인 시): 게시 완료 ✓ https://blog.naver.com/letter_hih/...
봇(게시 후): 이번 글 피드백 남겨주세요! (선택사항)
```

#### 댓글 봇 제어
```
/start_comment   댓글 봇 즉시 실행
/stop_comment    댓글 봇 중지
/status          오늘 댓글 현황 조회
```

#### 알림
- 댓글 봇 실행 완료 시 결과 요약 전송
- 오류 발생 시 즉시 알림

## 3. 기술 스택

| 역할 | 기술 |
|------|------|
| 오케스트레이션 | n8n (로컬, npx 실행) |
| AI 글·댓글 생성 | Claude API (claude-opus-4-6) |
| 브라우저 자동화 | Playwright (Python) |
| UI/입출력 | 텔레그램 봇 |
| 스타일 저장 | 로컬 마크다운 파일 |
| 이력 저장 | SQLite (기존 comments.db 확장) |

## 4. 파일/폴더 구조 (목표)

```
인성이프로젝트/
├── main.py                      # 댓글 봇 진입점 (기존)
├── publisher_main.py            # 게시물 발행 진입점 (신규)
├── config/settings.py           # 설정 (확장)
├── skills/
│   └── writing_style.md         # AI 글쓰기 스킬 (피드백 누적)
├── src/
│   ├── auth/                    # 로그인 (기존)
│   ├── collectors/              # 댓글 수집 (기존)
│   ├── commenter/
│   │   ├── comment_writer.py    # 댓글 작성 (기존, AI 방식으로 교체)
│   │   └── ai_comment.py        # AI 댓글 생성 (신규)
│   ├── publisher/               # 게시물 발행 (신규)
│   │   ├── blog_publisher.py    # 스마트에디터 조작
│   │   └── image_uploader.py    # 사진 업로드
│   ├── ai/                      # AI 연동 (신규)
│   │   ├── content_generator.py # 글 초안 생성
│   │   └── skill_manager.py     # 피드백 → 스킬 업데이트
│   └── storage/database.py      # DB (기존 + 확장)
└── n8n/
    └── workflows/
        ├── blog_post_flow.json  # 글 작성 워크플로
        └── comment_bot_flow.json # 댓글 봇 워크플로
```
