# 네이버 블로그 자동화 시스템 — 요구사항 정의서

> 작성일: 2026-03-16
> 프로젝트: 인성이프로젝트
> 버전: 2.0 (전면 재작성)

---

## 1. 시스템 개요

네이버 블로그 운영 전 과정을 자동화하는 **다중 사용자 SaaS**.
두 가지 핵심 기능을 **웹 대시보드**와 **텔레그램 봇** 두 채널에서 동등하게 제공한다.

| 기능 | 설명 |
|------|------|
| **게시물 작성 및 발행** | 사진+메모 → AI 초안 → 승인/수정/다시쓰기 → 네이버 자동 게시 |
| **댓글 자동 답방 봇** | 내 블로그 댓글 작성자 수집 → 방문 → AI 맞춤 댓글 작성 |

모든 사용자는 회원가입 후 **자신만의 독립적인** 게시물 발행 + 댓글 봇을 운영한다.

## 2. 핵심 기능

### 2-1. 게시물 작성 및 발행

#### 공통 플로우 (웹/텔레그램 동일)

```
사진 1장+ (필수) + 메모 (선택)
    ↓
Claude Vision 분석 → AI 초안 생성 (제목 + 본문 + 해시태그)
    ↓
초안 미리보기
    ↓
[✅ 게시] [✏️ 수정] [🔄 다시쓰기]
    ↓ (게시 선택 시)
네이버 블로그 자동 발행 (Playwright)
    ↓
발행 완료 알림 + URL 전달
```

#### 웹 플로우 (`/write` 페이지)

| 항목 | 구현 상태 |
|------|----------|
| 사진 업로드 (드래그&드롭, 최대 10장) | ✅ |
| 메모 입력, 카테고리 선택, 페르소나 선택 | ✅ |
| AI 초안 미리보기 (HTML 렌더링) | ✅ |
| **[발행] 버튼 → API `/publish` 호출 → 네이버 게시** | ❌ G1 |
| 피드백 입력 → 재생성 | ✅ |
| 발행 이력 표시 | ✅ |
| **발행 완료 알림 + URL** | ❌ G4 |

#### 텔레그램 플로우

| 항목 | 구현 상태 |
|------|----------|
| 사진 전송 (미디어 그룹 + 캡션 메모) | ✅ |
| AI 초안 → HTML 미리보기 파일 전송 | ✅ |
| **Inline Keyboard: [✅ 게시] [✏️ 수정] [🔄 다시쓰기]** | ❌ G2 |
| 게시: API `/publish` 호출 → 네이버 게시 → 완료 알림 + URL | ❌ G2 |
| 수정: 텍스트 입력 대기 → 피드백 반영 → 재생성 | ❌ G2 |
| 다시쓰기: 동일 사진으로 완전 새로 생성 | ❌ G2 |
| `/my_posts`: 최근 발행 이력 조회 | ❌ G2 |
| **발행 완료 알림 + URL** | ❌ G4 |

#### AI 글 생성 프로세스

1. 메모 파싱 (장소명, 시간, 감정 추출)
2. 사진 → Claude Vision 분석 (음식/장소/분위기)
3. 카테고리 자동 감지 (맛집/체험/재테크/리뷰/개인이벤트)
4. 페르소나 + 스타일 가이드 + few-shot 예시 기반 초안 생성
5. 해시태그 자동 생성 (15~25개, 3계층 구조)

> 구현 상태: ✅ 완료 (`src/ai/content_generator.py`)

#### 네이버 게시 방식

- Playwright로 스마트에디터 조작
- 사진 업로드 → 폰트 적용 → 제목/본문 입력 → 해시태그 → 게시
- **사용자별 쿠키/블로그ID로 발행** (다중 사용자) — ❌ G3

> 기본 발행 로직: ✅ 완료 (`src/publisher/blog_publisher.py`)
> 다중 사용자 지원: ❌ G3

### 2-2. 댓글 자동 답방 봇

#### 공통 동작

1. 내 블로그 댓글 작성자 수집 (관리자 페이지)
2. 오토 블로거 필터링 (패턴 분석 + 의심 점수)
3. 블로거 방문 → 최근 게시물 3개 → 본문 읽기
4. AI 맞춤 댓글 배치 생성 (Claude Haiku)
5. 타이핑 시뮬레이션 + 랜덤 딜레이로 작성
6. 실행 결과 기록 (SQLite + Supabase 이중)
7. 완료 알림 전송

> 구현 상태: ✅ 완료 (Phase 1)

#### 승인 모드

| 모드 | 설명 | 구현 상태 |
|------|------|----------|
| **수동 승인** | 댓글 생성 → 승인 대기 → 사용자 승인 → 작성 | ✅ |
| **자동 승인** | 댓글 생성 → 즉시 작성 (검증 후 전환) | ✅ |

#### 웹 제어 (`/bot` 페이지)

| 항목 | 구현 상태 |
|------|----------|
| 봇 실행(run) / 승인된 댓글 실행(execute) / 재시도(retry) 버튼 | ✅ |
| 승인 대기 댓글 목록 + 승인/거부 | ✅ |
| 실행 이력 + 통계 (성공률, 댓글 수) | ✅ |
| 봇 설정 변경 (시간대, 한도, 승인 모드) | ✅ |

#### 텔레그램 제어

| 항목 | 구현 상태 |
|------|----------|
| `/preview_comment {blog_id}`: 댓글 초안 미리보기 + [✅승인/❌거부/✏️수정] | ✅ |
| `/pending`: 승인 대기 목록 + Inline Keyboard | ✅ |
| `/execute`: 승인된 댓글 일괄 작성 | ✅ |
| `/status`: 오늘 현황 (댓글 수, 방문 블로거, 재시도 큐) | ✅ |
| `/settings`: 봇 설정 조회/변경 | ✅ |
| 실행 완료 시 결과 요약 텔레그램 알림 | ✅ |

#### 실행 시간대

| 구분 | 시간대 |
|------|--------|
| 평일 | 20:00~24:00 (기본) |
| 주말 | 13:00~18:00 |
| 커스텀 | 사용자별 설정 가능 |

#### 봇 감지 방지

- 글자 단위 타이핑 시뮬레이션
- 댓글 사이 랜덤 딜레이 (5~15초)
- 블로거 사이 랜덤 딜레이 (30~60초)
- AI 생성으로 댓글 다양성 확보
- 일일 한도 (블로거 10명, 댓글 30개)

## 3. 공통 요구사항

모든 사용자가 **게시물 발행 + 댓글 봇** 두 기능을 독립적으로 사용.

| 항목 | 구현 방식 | 구현 상태 |

| 쿠키 | 유저별 (`cookies/{user_id[:8]}_naver.json` + Supabase `bot_cookies`) | ✅ |

| 발행 API 다중 사용자 | `/publish`에서 user_id별 쿠키/블로그ID 사용 | ❌ G3 |

### G1: 웹 "네이버 발행" 버튼

**현황**: `/write` 페이지에 "저장" 버튼만 있음. `generation_queue`에 초안 저장은 되지만, 네이버 발행 호출 없음.

**수정 대상 파일**:

| 파일 | 변경 |
|------|------|
| `apps/web/app/(dashboard)/write/page.tsx` | "발행" 버튼 추가 (저장 옆) |
| `apps/web/app/api/publish/route.ts` | **신규** — Next.js API → FastAPI `/publish` 프록시 |

**구현 상세**:
- 발행 버튼 클릭 시:
  1. `generation_queue` 상태 → `publishing`
  2. Next.js API `/api/publish` 호출 (title, body, hashtags, image_paths, user_id)
  3. FastAPI `/publish` → Playwright 발행
  4. 성공: `generation_queue` 상태 → `published`, post_url 저장
  5. UI에 결과 표시 (URL 링크)

---

### G2: 텔레그램 게시물 승인/발행 플로우

**현황**: 사진 전송 → AI 초안 → HTML 미리보기 전송 → 끝. 승인/수정/다시쓰기 Inline Keyboard 없음.

**수정 대상 파일**:

| 파일 | 변경 |
|------|------|
| `telegram_bot.py` | `_process_photos()` 끝에 Inline Keyboard 추가 |
| `telegram_bot.py` | 콜백 핸들러 3개 추가 (publish/edit/regenerate) |

**구현 상세**:
- `_process_photos()` 응답에 Inline Keyboard 추가:
  ```
  [✅ 게시] [✏️ 수정] [🔄 다시쓰기]
  callback_data: "pub_approve:{queue_id}", "pub_edit:{queue_id}", "pub_regen:{queue_id}"
  ```
- `pub_approve` 콜백:
  1. `generation_queue`에서 해당 row 조회
  2. FastAPI `/publish` 호출 (해당 사용자 user_id 포함)
  3. 결과 메시지 전송 (성공 URL / 실패 이유)
- `pub_edit` 콜백:
  1. "수정 내용을 입력하세요" 프롬프트
  2. 다음 텍스트 메시지 → `/api/regenerate` 호출 (피드백 반영)
  3. 새 초안 → 다시 Inline Keyboard
- `pub_regen` 콜백:
  1. 동일 사진+메모로 재생성
  2. 새 초안 → 다시 Inline Keyboard

### G3: `/publish` API 다중 사용자

**현황**: `api_server.py`의 `/publish` 엔드포인트가 user_id를 받지 않음. 단일 쿠키 파일 사용.

**수정 대상 파일**:

| 파일 | 변경 |
|------|------|
| `api_server.py` | `PublishRequest`에 `user_id: str` 필드 추가 |
| `api_server.py` | 발행 로직에서 `get_user_bot_config(user_id)` 사용 |
| `src/publisher/blog_publisher.py` | `publish()` 메서드에 user_id 파라미터 전달 |
| `src/auth/naver_login.py` | 쿠키 로드 시 user_id별 경로 사용 |

**구현 상세**:
- 발행 흐름:
  1. `user_id`로 `bot_settings` → `naver_blog_id` 조회
  2. `user_id`로 `bot_cookies` → 쿠키 로드
  3. 해당 쿠키로 Playwright 로그인
  4. 해당 blog_id로 발행

### G4: 게시물 발행 완료 알림

**수정 대상 파일**:

| 파일 | 변경 |
|------|------|
| `api_server.py` | `/publish` 성공 후 `_send_telegram_notification()` 호출 |
| `api_server.py` | `PublishRequest`에 `chat_id: int | None` 추가 |
| `apps/web/app/(dashboard)/write/page.tsx` | 발행 API 응답 후 상태 UI 업데이트 |

**구현 상세**:
- 텔레그램: 발행 성공 시 해당 chat_id로 완료 메시지 + 블로그 URL 전송
- 웹: `/publish` API 응답의 post_url을 UI에 표시, `generation_queue` 상태 반영

### G5: 텔레그램 봇 다중 사용자

**현황**: 텔레그램 봇이 모든 명령을 단일 사용자(admin)로 처리.

**수정 대상 파일**:

| 파일 | 변경 |
|------|------|
| Supabase migration | `telegram_mappings` 테이블 추가 (telegram_chat_id ↔ user_id) |
| `telegram_bot.py` | `_get_user_id(chat_id)` 헬퍼 함수 추가 |
| `telegram_bot.py` | 모든 핸들러에서 user_id 조회 후 전달 |
| `telegram_bot.py` | `/link` 명령 추가 (텔레그램 ↔ 웹 계정 연결) |

**구현 상세**:
- 연결 플로우:
  1. 웹에서 "텔레그램 연결" → 1회용 코드 생성
  2. 텔레그램에서 `/link {코드}` → `telegram_mappings`에 매핑 저장
  3. 이후 모든 명령에서 chat_id → user_id 자동 조회
